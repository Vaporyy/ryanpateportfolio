Project 1

Foreign Exchange
Calculator

This program, on the surface, can seem quite simple, but the inner working can be quite complex.
The program utilizes different imported and base functions within python to create
a efficient and robust program.

With a plethora of uses, the foreign exchange calculator, whilst simple on the surface, employs looping, subroutines, import functions, Calculations etc. I decided to develop a foreign exchange calculator as I knew it would be complex and would test my skills and abilities to prepare myself for an exam I was sitting nearer the time. I used python to create this program and used a application called Pycharm as my preferred IDE to write my code. I tried to challenge myself by making the scenario more realistic by adding some requirements that I would have to meet. For example, an employee discount that would be added to the final total.

The development process started by setting myself particular goals. These goals consisted of:

  • Writing a receipt to the user
  • Having the user be able to select between 5 different currencies that they can convert to
  • Have a transaction fee added depending on the amount of GBP the user wants to exchange
  • A max conversion rate
  • An employee discount that can be added when selected by the user.
I wanted to have the program be fully functional and as efficient as possible. I started by planning using psuedocode in a word document and decided I would need a main loop which would enclose 4 "sub-loops" each with their own purpose and with included validation. To keep it simple, I gave them each one letter names of V (for the main loop), W, X, Y and Z. V, as stated, is the main loop which contains the other 4 loops. Its main purpose though is to keep the program operational as long as its left on. W is the introduction, where the user will be asked if they want to use the converter. They can then either say "yes (Y)" or "no (N)" which will then either allow them to move on to "X" or give a goodbye message. X asks the user for the amount of GBP they would like to exchange. This loop includes one of the first goals I wanted to achieve which was to include a max conversion rate to which I set to "2500". If the amount input is less it will then move on to "Y". Y is where the main calculations take place. It allows the user to convert to "DOLLARS (USD), EUROS (EUR), REAL (BRL), YEN (JPY) and LIRA (TRY)" which then meets one of my other goals of having 5 different currencies to convert to. Once they have chosen, the program then moves to the final loop, Z. Z is where the user will be asked if they are a employee or not which will calculate a discount that will be added onto the final total which again meets one of the goals I set of having an employee discount. The receipt is created via a subroutine which is then called at the end of the program under the name of "receipt()", which will allow the user to have a printable and savable copy of their conversion. The transaction fee is calculated outside the "sub-loops" and inside the main V loop. It simply just compares the amount of GBP the user input to find the fee that would need to be added on top of their final total.

When I finished with design and started coding, I realised I wanted to have the program to have more of a personal touch. So I imported the time, datetime and random function. I used the time function to allow stops between some of the outputs so its easier for the user to read. I also used it and the random function to create a subroutine which made it look like the program actually needed time to calculate (mainly included just for aesthetics). I also used it again for the "type" subroutine which would make it look like the computer was typing the outputs out to you as the user. The datetime function was just used to display the date and time at the top of the program to make it seem more realistic (again just for aesthetics).
